# Almaz Tour — сайт туристического агентства

Современный сайт агентства Almaz Tour (Астана): Next.js 16 (App Router) + TypeScript + Tailwind CSS 4. Полный редизайн almaztour.kz с сохранением всех реальных данных: контактов, отзывов, текстов направлений и интеграции с Tourvisor.

## Запуск

```bash
npm install
npm run dev        # разработка на http://localhost:3000
npm run build      # продакшен-сборка
npm start          # запуск собранного сайта
```

Требуется Node.js 20+.

## Структура

```
src/
  app/                    страницы (App Router)
    page.tsx              главная
    hot-tours/            горящие туры (модуль Tourvisor)
    destinations/         каталог 15 стран + страницы направлений
    health-tours/         оздоровительные туры
    reviews/              отзывы + форма отзыва
    contacts/             контакты + форма «Подобрать тур»
    api/                  приём форм (lead, review, subscribe)
    sitemap.ts, robots.ts SEO
  components/             компоненты (шапка, футер, карточки, формы, Tourvisor)
  data/                   контент: site.ts, destinations.ts, reviews.ts
public/images/            логотип и фотографии направлений
docs/competitor-analysis.md  анализ конкурентов
```

## Tourvisor

Поиск туров и горящие туры подтягиваются встраиваемыми модулями Tourvisor.
ID модулей задаются в `.env.local` (пример — в `.env.example`):

```
NEXT_PUBLIC_TV_SEARCH_ID=...     # модуль «Поиск туров»
NEXT_PUBLIC_TV_HOT_HOME_ID=...   # «Горящие туры» на главной
NEXT_PUBLIC_TV_HOT_PAGE_ID=...   # «Горящие туры» на /hot-tours
```

Где взять свои ID:

1. Зарегистрируйтесь или войдите в кабинет партнёра: https://pro.tourvisor.ru
2. Раздел «Модули для сайта» → создайте модуль нужного типа
   («Поиск туров», «Горящие туры»).
3. В коде вставки модуля будет строка вида `tv-moduleid-9955693` —
   число из неё и есть ID.
4. Там же в кабинете настраиваются город вылета по умолчанию (Астана),
   цвета и поля модуля — подгоните их под палитру сайта:
   тёмно-синий `#000942`, золотой `#FFCC00`, лазурный `#5DA8D3`.

Важно: модули привязываются к домену сайта в кабинете Tourvisor, поэтому
на localhost виджет может не отрисоваться. В этом случае компонент
`TourvisorModule` показывает запасной блок с кнопкой WhatsApp — это
ожидаемое поведение, на боевом домене виджет заработает.

Ещё нюанс: виджет сам определяет город вылета по IP посетителя. Если при
разработке вы видите «из Перми» и цены в рублях — это гео-детект, а не
ошибка. Город вылета по умолчанию (Астана) и валюту жёстко фиксируют в
настройках модуля в кабинете pro.tourvisor.ru.

## Формы

Формы «Подобрать тур», «Оставить отзыв» и подписка валидируются на клиенте
и отправляются в API-роуты (`src/app/api/*`). Каждая заявка пишется в
серверный лог (дубль-резерв) и мгновенно уходит сообщением в Telegram —
отправку делает общий хелпер `src/lib/telegram.ts`.

### Подключение Telegram

1. **Создайте бота.** Напишите [@BotFather](https://t.me/BotFather) в
   Telegram → команда `/newbot` → задайте имя и username бота. BotFather
   выдаст токен вида `1234567890:AA...` — это `TELEGRAM_BOT_TOKEN`.
2. **Узнайте chat_id.** Напишите своему боту любое сообщение (бот не может
   написать первым), затем откройте в браузере
   `https://api.telegram.org/bot<ТОКЕН>/getUpdates` — в ответе будет
   `"chat":{"id":123456789,...}`, это число и есть `TELEGRAM_CHAT_ID`.
   Чтобы заявки приходили в группу, добавьте бота в группу и возьмите её id
   оттуда же (у групп он отрицательный, вида `-100...`).
3. **Задайте переменные.** Локально — в `.env.local`:

   ```
   TELEGRAM_BOT_TOKEN=1234567890:AA...
   TELEGRAM_CHAT_ID=123456789
   ```

   На Vercel — Settings → Environment Variables, те же два имени
   (для Production и Preview), затем передеплойте проект.

Это серверные секреты: префикс `NEXT_PUBLIC_` не нужен, в клиентский бандл
они не попадают. Если переменные не заданы, сайт работает как раньше —
заявки просто пишутся в серверный лог. Если Telegram недоступен, роут
отвечает 502, и форма показывает посетителю запасное сообщение с просьбой
позвонить.

## Деплой

Проще всего — Vercel:

```bash
npx vercel
```

Не забудьте добавить переменные из `.env.local` в настройки проекта
(Settings → Environment Variables). Подойдёт и любой сервер с Node.js:
`npm run build && npm start` за реверс-прокси.
